# Internal Documentation (Tracked)

This folder contains **tracked** internal documentation - valuable domain knowledge that should be committed to the repository.

## Structure

- **`knowledge/`** - Domain knowledge base
  - `sec-forms/` - SEC form specifications and parsing research
  - `xbrl/` - XBRL parsing knowledge and architecture
  - `patterns/` - Common patterns and solutions
  - `architecture/` - System design documentation

## Guidelines

Content here should:
1. Help future developers (human or AI) understand the SEC/XBRL domain
2. Document decisions and their rationale
3. Be reusable across multiple development sessions
4. Not contain sensitive business information

## Related Locations

- **`docs-internal/`** - Private, gitignored docs for temporary research and planning
- **`docs/`** - Public user-facing documentation
- **`edgar/xbrl/CLAUDE.md`** - XBRL package guide
